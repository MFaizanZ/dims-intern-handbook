What each is for: Git manages code history (ch 04). pnpm is a fast package manager for the JavaScript/Next.js side. uv is a fast Python environment/package manager. Docker runs the supporting services (database, message broker, storage) in isolated containers so everyone’s setup matches. Running the platform (the usual shape)

The backend depends on a few services — the MQTT broker (EMQX), PostgreSQL, MinIO (storage) and Redis. Docker brings them all up together:

# 1) bring up the supporting stack (EMQX + Postgres + MinIO + Redis)
pnpm stack:up

# 2) run the bridge + operator API (the backend, ch 13)
cd services/dji-bridge
uv run --env-file ../../.env python -m dji_bridge.operator

# 3) in another terminal, run the web console (ch 14)
pnpm dev

Then open the console in your browser. The first time, you copy .env.example to .env and fill in local passwords + your DJI App credentials.

Developing without a drone — the simulator

You almost never need real hardware. The SITL simulator role-plays a Dock 3 + M4TD over the same broker (ch 13), so the whole pipeline — telemetry, a real flight model, media upload, AI, alerts — runs end-to-end on your laptop:

# start the virtual dock (only when NOT connected to a real dock)
uv run --env-file scripts/demo/demo.env python -m dji_bridge.simulator
⚠ Safety rule

Do not start the simulator (or push test schedules) while the backend is bound to a real Dock 3 — the real drone must never launch by accident. There’s an “armed” master gate in the UI (ch 14) that defaults to OFF for exactly this reason. When in doubt, stay in demo mode.

A couple of gotchas worth knowing early

GPU inference can silently fall back to CPU

As covered in chapter 15, a plain uv run can revert the Python env to the CPU-only build of onnxruntime, dropping AI inference from ~78 fps to ~12. If inference is suddenly slow, run the GPU repair script and use the --no-sync flag. This trips people up — now you know.

Don’t kill every Node process to restart the dev server

Stopping all Node processes also kills tooling that rides on Node (like the browser-automation used for screenshots). To restart the web console, stop only the dev task / its port — not every Node process on the machine.
